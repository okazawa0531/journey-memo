# 操作マニュアル - 旅行メモアプリ

## 利用者向け操作方法

### 1. ログイン

1. ブラウザで CloudFront URL（例: `https://xxxxxxxxxxxxxxx.cloudfront.net`）にアクセスします
2. ログイン画面が表示されます
3. パスワード入力欄に **`okazawa`** と入力します
4. 「ログイン」ボタンをクリックします
5. 認証が成功すると、日本地図画面に移動します

**注意**: パスワードを間違えると「パスワードが正しくありません」と表示されます。再度正しいパスワードを入力してください。

---

### 2. 日本地図の見方

日本地図画面では、47都道府県が地図上に表示されています。

| 色 | 意味 |
|----|------|
| オレンジ色 | 訪問済みの都道府県 |
| 白色 | 未訪問の都道府県 |

画面上部に「X / 47 都道府県」と訪問済み都道府県数が表示されます。

都道府県にマウスカーソルを合わせると、都道府県名がツールチップで表示されます（スマートフォンではタップで選択できます）。

---

### 3. 都道府県をクリックして記録を編集する

1. 地図上の都道府県をクリック（スマートフォンではタップ）します
2. 編集モーダルが開きます

#### モーダルの操作方法

**訪問済みチェックボックス**
- チェックを入れると、地図上でオレンジ色に変わります
- 「保存」ボタンを押すと反映されます

**評価（星）**
- ★ をクリックすることで 1〜5 の評価を付けられます
- 同じ星を再度クリックすると評価がリセットされます

**訪問日の追加**
1. 日付入力フィールドをクリックし、訪問した日付を選びます
2. 「追加」ボタンをクリックします
3. 複数の日付を追加できます（訪問するたびに記録できます）
4. 削除したい日付の横にある「✕」をクリックすると削除できます

**メモ**
- 旅行の感想、おすすめスポット、食べたものなど、自由にメモを記入できます

**保存 / キャンセル**
- 「保存」をクリックすると変更が保存されます
- 「キャンセル」をクリックすると変更を破棄してモーダルを閉じます

---

### 4. ログアウト

画面右上の「ログアウト」ボタンをクリックすると、ログアウトしてログイン画面に戻ります。

---

### 5. スマートフォンでの操作

本アプリはスマートフォンに対応しています。
- 地図はタップで都道府県を選択できます
- モーダルはスクロール可能です
- 日付入力はスマートフォンのカレンダーが表示されます

---

## 管理者向け情報

### AWSコンソールでのデータ確認

旅行記録は DynamoDB テーブル `journey-memo-travels` に保存されています。

1. [AWS マネジメントコンソール](https://console.aws.amazon.com/) にログインします
2. DynamoDB サービスに移動します
3. 「テーブル」→「journey-memo-travels」をクリックします
4. 「テーブルアイテムの探索」タブでデータを確認できます

#### データ構造

各アイテムは以下の形式で保存されます:

```json
{
  "prefectureCode": "13",        // 都道府県コード（2桁）
  "visited": true,               // 訪問済みフラグ
  "notes": "素敵な旅行でした",    // メモ
  "visitDates": ["2024-01-01"],  // 訪問日リスト
  "rating": 5,                   // 評価（0-5）
  "updatedAt": "2024-01-01T12:00:00.000Z"  // 更新日時
}
```

### パスワードの変更方法

パスワードは Lambda 関数の環境変数または CDK スタック設定で管理されています。

#### 方法1: AWS マネジメントコンソールから変更

1. Lambda サービスに移動します
2. 関数 `JourneyMemoStack-AuthFunction-XXXXXXXX` を開きます
3. 「設定」タブ→「環境変数」を開きます
4. 環境変数に `CORRECT_PASSWORD` は現在コードにハードコードされているため、Lambda の更新が必要です

#### 方法2: コードを修正して再デプロイ

`backend/src/auth.ts` の以下の行を変更します:

```typescript
const CORRECT_PASSWORD = 'okazawa'  // ← ここを変更
```

変更後、バックエンドを再ビルドして CDK で再デプロイします:

```bash
cd backend && npm run build
cd ../cdk && npx cdk deploy
```

#### JWT シークレットの変更（推奨）

本番運用では `cdk/lib/journey-memo-stack.ts` の JWT_SECRET を安全な値に変更することを強く推奨します:

```typescript
const lambdaEnv = {
  TABLE_NAME: table.tableName,
  JWT_SECRET: 'your-secure-random-secret-here',  // ← 強力なランダム文字列に変更
}
```

または AWS Secrets Manager を使用して安全に管理することを推奨します。

### CloudWatch ログの確認

Lambda 関数のログは CloudWatch Logs で確認できます:

1. CloudWatch サービスに移動します
2. 「ロググループ」から `/aws/lambda/JourneyMemoStack-AuthFunction-XXXXXXXX` または
   `/aws/lambda/JourneyMemoStack-TravelsFunction-XXXXXXXX` を選択します
3. 最新のログストリームでエラーや実行ログを確認できます
