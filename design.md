# 概要
旅行先を記録するためのアプリケーションです。

# 構成
このアプリケーションは以下の構成とします。

|構成要素|役割|
|----|----|
|フロントエンド|一般的なフロントエンドの役割です。主にUIを担当します。ログインページと旅行先画面ページの2つを提供します。|
|バックエンド|一般的なバックエンドの役割です。主にDBの操作を担当します。|

# 機能仕様
以下の機能を提供するアプリケーションです。
|機能|説明|
|----|----|
|ログイン画面|パスワードに"okazawa"と入力すると旅行先画面に遷移するだけのシンプルな認証画面です。ユーザ名も不要です。|
|旅行先画面|旅行先を記録・閲覧します。日本地図を表示し都道府県をクリックすることにより、その都道府県での旅行の記録を編集・閲覧できます。日本地図のうち、旅行先が記録されている都道府県はオレンジ色になり、記録されていない都道府県は白にしてください。|

上記は最低限の機能です。こういう機能もあるといいと思ったものは入れてください。
スマートフォンでの利用も可能にしてください。

# 環境
AWS環境で動作させます。CDKによる構築環境も作成してください。

## 環境に対する要求
- 可能な限りコストを安く抑えてください。デプロイしても使用しなければ料金は発生しないことがベストです。発生しなければ構築できない場合は、発生するコストの目安を表示してください。
- DBはDynamoDBを使用してください。最初の要求のため、キャパシティユニットはオンデマンドでお願いします。

# デザイン
目に優しい緑基調でお願いします。
